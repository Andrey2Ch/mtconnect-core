# Инструкция по локальному запуску проекта MTConnect

Этот документ описывает, как запустить все компоненты системы на локальной машине для разработки и отладки.

Вам понадобится **два окна терминала**.

---

### Шаг 1: Запуск Cloud API (Сервер, API, База Данных)

Это "мозг" системы. Он принимает данные, сохраняет их в MongoDB и отдает их дашборду.

1.  Откройте **первый терминал**.
2.  Перейдите в папку с Cloud API и запустите его. Эта команда запустит сервер на `http://localhost:3000`.

    ```powershell
    cd apps/cloud-api; npm run start:dev
    ```
3.  Оставьте этот терминал работать. В нем будут появляться логи о получении данных.

---

### Шаг 2: Запуск Edge Gateway (Сборщик данных)

Это "сборщик". Он опрашивает MTConnect агенты и ADAM-счетчики, а затем отправляет собранные данные в Cloud API.

1.  Откройте **второй терминал**.
2.  Убедитесь, что вы находитесь в корневой папке проекта (`C:\Projects\MTConnect`).
3.  Запустите Edge Gateway следующей командой:

    ```powershell
    npx ts-node src/main.ts
    ```
4.  Эта команда использует `ts-node` для компиляции и запуска TypeScript "на лету", используя `config.local.json` для настроек. В терминале вы увидите логи подключения к оборудованию и отправки данных.

---

### Шаг 3: Проверка результата

1.  После запуска обоих сервисов, откройте в вашем браузере дашборд по адресу:
    *   **`http://localhost:3000/dashboard-new.html`**

2.  Вы должны увидеть все 18 машин (8 MTConnect + 10 ADAM) со статусом "online" и актуальными данными.
